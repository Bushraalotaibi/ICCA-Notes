# Cloud Identity, Security, and Compliance (ICCA)

This section covers foundational security concepts in cloud, with a big focus on IAM and basic protection/compliance ideas.

## Topics Covered
- Identity and Access Management (IAM):
  - Users, groups, roles
  - Policies (permissions)
  - Least privilege principle
- Authentication & authorization basics
- Security foundations:
  - Why cloud security is different (shared responsibility)
  - Basic infrastructure protection concepts (high-level)
- Compliance and governance (high-level):
  - Why regulations matter
  - Basic controls and auditing concepts

## High-Yield Notes (Exam Focus)
- Understand the difference between:
  - **User vs Role**
  - **Authentication vs Authorization**
  - **Group vs Policy**
- Always apply **least privilege**
- Recognize common identity risks:
  - Over-permissive policies
  - Exposed credentials
  - Misconfigured access

## Practical Mini Examples
- Use a **role** when a service needs permissions (instead of hardcoding keys)
- Use **groups** to manage multiple users consistently
